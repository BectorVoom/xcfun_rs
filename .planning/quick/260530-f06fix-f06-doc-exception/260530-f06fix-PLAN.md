# Quick Task 260530-f06fix: F-06 documented-exception thresholds + beckesrx erf exclusion

## Problem

Two F-06 CI validation sweeps went red (order-3: run 26668931715 → 0/29; potential:
run 26668932207 → 30/32). Investigation (read-only, complete) showed the failures are
**ULP-accumulation drift** (order-3 GGAs, verdict-counting max ≤ 7.5e-9) plus
**erf_precise cancellation** (beckesrx/beckecamx), **not correctness bugs** — orders 0-2
are bit-exact and the `F::new` f32 pitfall is already fully fixed (zero `F::new(` calls
remain in any kernel). Per-functional verdict-counting max rel_err (excluding
below-clamp `excluded_by_regularize_clamp_design` diagnostic records) was measured from
the CI `report.jsonl` artifacts.

User-approved resolution: the **documented-exception (D-19/D-24) pattern** — tight,
per-functional, data-justified threshold overrides (no blanket relaxation, per D-18),
plus an erf-cancellation exclusion for beckesrx in PartialDerivatives mode.
