/// Number of elements in a multivariate Taylor expansion.
/// C(n_vars + order, order) computed iteratively.
pub const fn taylorlen(n_vars: usize, order: usize) -> usize {
    let mut len: usize = 1;
    let mut k: usize = 1;
    while k <= order {
        len = len * (n_vars + k) / k;
        k += 1;
    }
    len
}

2. Create crates/xcfun-core/src/density_vars.rs:

DensityVars<T: Num> struct with 25 fields as specified in docs/design/01-data-structures.md section 1. All fields pub.

Implement `from_input(input: &[T], var_type: VarType) -> Self` per D-12 and C++ densvars.hpp:

The construction pattern for each VarType (translate C++ switch-case fallthrough to explicit Rust match arms):

For alpha/beta types (A, A_B, A_GAA, A_B_GAA_GAB_GBB, etc.):
- Extract a, b (b=zero for unpolarized types like A, A_GAA)
- Regularize a and b: if a.c[0] < TINY_DENSITY, set a.c[0] = TINY_DENSITY (for CTaylor) or a = TINY_DENSITY (for f64). Same for b.
- Compute n = a + b, s = a - b

For total/spin types (N, N_S, N_GNN, N_S_GNN_GNS_GSS, etc.):
- Extract n, s (s=zero for unpolarized types like N, N_GNN)
- Regularize n
- Compute a = (n + s) * 0.5, b = (n - s) * 0.5
- Regularize a and b independently

For gradient types:
- Extract gaa, gab, gbb (or gnn, gns, gss)
- Compute cross-conversions: gnn = gaa + 2*gab + gbb, gns = gaa - gbb, gss = gaa - 2*gab + gbb (or reverse)

For gradient-component types (A_AX_AY_AZ, etc.):
