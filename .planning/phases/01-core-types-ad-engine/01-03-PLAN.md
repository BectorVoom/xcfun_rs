Create crates/xcfun-ad/src/tmath.rs implementing all Taylor expansion generators from C++ tmath.hpp. Each function fills a coefficient array t[0..=degree] with the Taylor coefficients of the function around point x0.

ALL expansion functions take `(t: &mut [f64], x0: f64)` where t.len() - 1 is the degree.

1. `pub fn exp_expand(t: &mut [f64], x0: f64)`:
   t[0] = x0.exp(); for i in 1..t.len(): t[i] = t[0] / factorial(i)
   where factorial is computed iteratively: ifac *= i as f64

2. `pub fn log_expand(t: &mut [f64], x0: f64)`:
   t[0] = x0.ln(); inv = 1.0/x0; sign = 1.0;
   for i in 1..t.len(): t[i] = sign * inv / (i as f64); inv /= x0; sign = -sign;
   Actual formula: t[k] = (-1)^(k+1) / (k * x0^k)

3. `pub fn pow_expand(t: &mut [f64], x0: f64, a: f64)`:
   t[0] = x0.powf(a);
   for i in 1..t.len(): t[i] = t[i-1] * (a - (i-1) as f64) / (i as f64 * x0)
   This is the falling factorial recurrence.

4. `pub fn sqrt_expand(t: &mut [f64], x0: f64)`:
   Special case of pow_expand with a=0.5. Can delegate or use the optimized recurrence:
   t[0] = x0.sqrt(); x0inv = 1.0/x0;
   for i in 1..t.len(): t[i] = t[i-1] * (1.5 / (i as f64) - 1.0) * x0inv
   Verify this matches pow_expand(t, x0, 0.5).

5. `pub fn cbrt_expand(t: &mut [f64], x0: f64)`:
   t[0] = x0.cbrt(); x0inv = 1.0/x0;
   for i in 1..t.len(): t[i] = t[i-1] * (4.0 / (3.0 * i as f64) - 1.0) * x0inv

6. `pub fn inv_expand(t: &mut [f64], x0: f64)`:
   t[0] = 1.0/x0;
   for i in 1..t.len(): t[i] = -t[i-1] / x0

7. `pub fn sin_expand(t: &mut [f64], x0: f64)` and `pub fn cos_expand(t: &mut [f64], x0: f64)`:
   Interleaved computation. sin coefficients: t[0]=sin(x0), t[1]=cos(x0), t[2]=-sin(x0)/2!, t[3]=-cos(x0)/3!, ...
   Pattern: t[k] = { sin(x0)*(-1)^(k/2)/k! if k even, cos(x0)*(-1)^((k-1)/2)/k! if k odd } but shifted...
   
   Actually from C++ tmath.hpp, the expansion uses:
   s = sin(x0), c = cos(x0), ifac = 1.0
   for k in 0..=degree:
     match k % 4:
       0 => t[k] = s / ifac
       1 => t[k] = c / ifac  
       2 => t[k] = -s / ifac
       3 => t[k] = -c / ifac
     ifac *= (k+1) as f64

   Verify against C++ source.

8. `pub fn atan_expand(t: &mut [f64], x0: f64)`:
   Multi-step: first expand 1/(1+x0^2), then compose with 2*x0*h + h^2, then integrate.
   From C++ tmath.hpp: 
   - Compute g = 1/(1+x0^2) via pow_expand with a=-1 applied to (1+x0^2)
   - The 1D composition with the polynomial 2*x0*h + h^2 uses taylor1d_compose
   - Integrate the result to get atan coefficients
   - Set t[0] = atan(x0)
   
   Implement the 1D compose function: `pub fn taylor1d_compose(result: &mut [f64], f: &[f64], g_coeffs: &[f64])` that computes f(g(h)) as a Taylor series. Use Horner evaluation.

9. `pub fn erf_expand(t: &mut [f64], x0: f64)`:
   Multi-step from C++ tmath.hpp:
   - Compute Gaussian: exp(-x0^2) expanded around x0
   - Multiply by 2/sqrt(pi)
   - Compose with polynomial (-2*x0*h - h^2) for the exp(-x^2) = exp(-(x0+h)^2) expansion
   - Integrate to get erf coefficients
   - Set t[0] = erf(x0) (use the f64 erf approximation from Num trait)

10. `pub fn asinh_expand(t: &mut [f64], x0: f64)`:
    Multi-step: d/dx asinh(x) = 1/sqrt(1+x^2)
    - Compute (1+x0^2)^(-1/2) via pow_expand
    - Compose with 2*x0*h + h^2
    - Integrate
    - Set t[0] = asinh(x0)

11. `pub fn asin_expand(t: &mut [f64], x0: f64)`:
    d/dx asin(x) = 1/sqrt(1-x^2)
    - pow_expand of (1-x0^2) with a=-0.5
    - Compose with -2*x0*h - h^2
    - Integrate
    - t[0] = asin(x0)

12. `pub fn acos_expand(t: &mut [f64], x0: f64)`:
    d/dx acos(x) = -1/sqrt(1-x^2), so negate asin_expand result except t[0] = acos(x0)

13. `pub fn sqrtx_asinh_sqrtx_expand(t: &mut [f64], x0: f64)`:
    Per D-11: uses Pade approximant near x=0 matching C++ implementation.

    Otherwise: compute sqrt(x0) * asinh(sqrt(x0)) and its derivatives.

    - For large x: compute s = sqrt(x0), a = asinh(s), f = s*a
    - Expand derivatives using the identity: f'(x) = asinh(sqrt(x))/(2*sqrt(x)) + 1/(2*(1+x))
    - For small x: Pade approximant with hardcoded coefficients

14. Integration helper: `fn integrate(t: &mut [f64])`:
    Shift coefficients right by one position and divide by index:
    for i in (1..t.len()).rev(): t[i] = t[i-1] / (i as f64)
    t[0] = 0.0 (caller sets the integration constant)

pub fn ctaylor_exp<const N: usize>(x: CTaylor<f64, N>) -> CTaylor<f64, N> {
    let x0 = x.c[0];
    let degree = N; // number of variables = max derivative order for composition
    let mut coeffs = vec![0.0f64; degree + 1];
    tmath::exp_expand(&mut coeffs, x0);
    let mut result = CTaylor::<f64, N>::zero_array();
    compose::compose(&mut result, &x.c, &coeffs);
    CTaylor { c: result }
}

But wait -- N here is the number of CTaylor variables, and the composition needs coefficients up to degree N. So the expansion functions must produce N+1 coefficients (degree 0 through N). The expansion functions from Task 1 take `t: &mut [f64]` where t.len() = degree + 1. So pass a slice of length N+1.

HOWEVER: There is a subtlety. The C++ compose() function in ctaylor.hpp takes coefficients of length equal to the number of VARIABLES (Nvar), not the polynomial degree. The Horner loop runs from index N-1 down to 0, applying multo_skipconst N times. So the coefficients array length is N+1.
