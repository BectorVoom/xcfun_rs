4. Create crates/xcfun-ad/src/compose.rs implementing the recursive algorithms from C++ ctaylor.hpp:

- `pub fn mul_recursive(dst: &mut [f64], a: &[f64], b: &[f64])`: Accumulating recursive multiply. Base case n==1: dst[0] += a[0]*b[0]. Recursive: mul_recursive(dst[..half], a[..half], b[..half]); mul_recursive(dst[half..], a[half..], b[..half]); mul_recursive(dst[half..], a[..half], b[half..]).

- `pub fn mul_set_recursive(dst: &mut [f64], a: &[f64], b: &[f64])`: Non-accumulating first call. Base case n==1: dst[0] = a[0]*b[0]. Recursive: mul_set(dst[..half], a[..half], b[..half]); mul_set(dst[half..], a[half..], b[..half]); mul(dst[half..], a[..half], b[half..]).

- `pub fn multo_skipconst(dst: &mut [f64], y: &[f64])`: In-place multiply by (y - y[0]). CRITICAL ordering: process high half first (reads low half before modification), then add cross terms, then process low half. Base case n==1: dst[0] = 0.0. For the cross-term step where we need simultaneous read of dst[..half] and write to dst[half..], copy dst[..half] to a stack-allocated temporary first to satisfy the borrow checker.

- `pub fn compose(result: &mut [f64], poly: &[f64], coeffs: &[f64])`: Horner composition. result[0] = coeffs[last]; result[1..] = 0; for i = (coeffs.len()-2) downto 0: multo_skipconst(result, poly); result[0] += coeffs[i]. The coeffs slice has length = number of variables N + 1. MUST match C++ ctaylor_rec::compose exactly.

CRITICAL for multo_skipconst borrow checker issue: The function reads dst[..half] while writing dst[half..]. Use `let lo_copy: Vec<f64> = dst[..half].to_vec();` then `mul_recursive(&mut dst[half..], &lo_copy, &y[half..]);`. The max copy size is 64 f64s (512 bytes) at N=7, which is negligible.

5. Implement CTaylor division in ctaylor.rs: Division by CTaylor q computes 1/q via compose with inverse Taylor coefficients [1/q0, -1/q0^2, 2/q0^3, -6/q0^4, ...] = (-1)^k * k! / q0^(k+1), then multiplies. Use the inv_expand pattern: t[0] = 1/x0, t[k] = -t[k-1]/x0.

Create crates/xcfun-ad/src/num.rs with the Num trait definition per docs/design/02-traits.md:

```rust
pub trait Num:
    Clone
    + std::ops::Add<Output = Self>
    + std::ops::Sub<Output = Self>
    + std::ops::Mul<Output = Self>
    + std::ops::Div<Output = Self>
    + std::ops::Neg<Output = Self>
    + std::ops::AddAssign
    + std::ops::SubAssign
    + std::ops::MulAssign
    + Sized
{
    fn from_f64(val: f64) -> Self;
    fn zero() -> Self;
    fn one() -> Self;

    /// Get the scalar (constant-term) value as f64.
    /// For f64: returns self.
    /// For CTaylor: returns c[0].
    /// Required by DensityVars regularization (D-12) to inspect the constant term.
    fn value_f64(&self) -> f64;

    /// Set only the constant term, preserving all derivative coefficients.
    /// For f64: sets self = val.
    /// For CTaylor: sets self.c[0] = val, leaving c[1..] unchanged.
    /// Required by DensityVars regularization (D-12) to clamp density without
    /// destroying derivative information.
    fn set_constant(&mut self, val: f64);

    fn exp(self) -> Self;
    fn log(self) -> Self;
    fn sqrt(self) -> Self;
    fn cbrt(self) -> Self;
    fn pow(self, exponent: f64) -> Self;
    fn powi(self, n: i32) -> Self;
    fn abs(self) -> Self;

    fn sin(self) -> Self;
    fn cos(self) -> Self;
    fn atan(self) -> Self;
    fn asin(self) -> Self;
    fn acos(self) -> Self;

    fn asinh(self) -> Self;
    fn erf(self) -> Self;
    fn sqrtx_asinh_sqrtx(self) -> Self;

    fn lt(&self, other: &Self) -> bool;
    fn gt(&self, other: &Self) -> bool;
}
```

- from_f64(val) -> val
- zero() -> 0.0, one() -> 1.0
- value_f64(&self) -> *self (trivial: the f64 IS the scalar value)
- set_constant(&mut self, val: f64) -> *self = val (trivial: replace the value)
- exp -> f64::exp(), log -> f64::ln(), sqrt -> f64::sqrt(), cbrt -> f64::cbrt()

  ```
  erf(x) = 1 - (a1*t + a2*t^2 + a3*t^3 + a4*t^4 + a5*t^5) * exp(-x^2)

  ```
  Handle sign: erf(-x) = -erf(x)

- lt/gt -> standard f64 comparison
