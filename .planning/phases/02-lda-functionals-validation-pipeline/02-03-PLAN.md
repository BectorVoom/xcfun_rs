**set(&mut self, name: &str, weight: f64) -> Result<(), XcError>** (per D-18):
1. Check `aliases::lookup(name)` first
2. If alias found: for each `(id, w)` in alias.terms, call `self.add_functional(FunctionalImpl::from_id(*id), weight * w)`
3. If not alias: try `FunctionalId::from_name(name)`. If found, `self.add_functional(FunctionalImpl::from_id(id), weight)`
4. If neither: return `Err(XcError::UnknownName(name.to_string()))`
5. Private helper `add_functional(&mut self, f: FunctionalImpl, w: f64)`: push to `self.active`, update `self.depends |= f.depends()`

**eval_setup(&mut self, vars: VarType, mode: EvalMode, order: u32) -> Result<(), XcError>** (per D-13):
1. Validate: `if self.active.is_empty() { return Err(XcError::NotConfigured) }`
2. Validate: `if order > MAX_ORDER { return Err(XcError::InvalidOrder { order, mode, n_vars: vars.input_len() }) }`
3. Validate: `if !vars.provides().contains(self.depends) { return Err(XcError::InsufficientVars { vars, required: self.depends }) }`
4. Validate mode compatibility: Potential mode requires depends to NOT include LAPLACIAN or KINETIC (meta-GGA not supported for potential). If `mode == Potential && self.depends.intersects(Dependency::LAPLACIAN | Dependency::KINETIC) { return Err(XcError::UnsupportedMode { mode, depends: self.depends }) }`
5. Store `self.vars = Some(vars)`, `self.mode = Some(mode)`, `self.order = Some(order)`
6. Compute `self.input_len = vars.input_len()`
7. Compute `self.output_len` based on mode:
   - PartialDerivatives: `taylorlen(vars.input_len(), order as usize)` -- NOTE: the C++ uses the number of INPUT variables as the differentiation dimension
   - Potential: for LDA = 1 (energy) + number of spin channels (1 for closed-shell, 2 for A_B); for GGA = same + gradient contributions. For now: if A or N vars => 2 outputs (E, v_xc); if A_B or N_S vars => 3 outputs (E, v_alpha, v_beta).

**PartialDerivatives mode** (per D-14, EVAL-02):

CTaylor<f64, N> with N = 1 << n_vars provides first-order multilinear derivatives for n_vars variables. The coefficient layout for N=4 (2 vars) is: `[constant, var0, var1, var0*var1]` -- these are indices 0, 1, 2, 3. This gives energy + first derivatives + first-order mixed derivative.

For higher derivative orders, the C++ xcfun uses a **variable-pair loop**: iterate over all pairs (i,j) with i<=j, seeding a fresh CTaylor evaluation for each pair. Each evaluation produces the mixed partial derivative d^2E/dx_i dx_j. The output array has taylorlen(n_vars, order) entries arranged in the standard Taylor expansion order.

**Implementation approach (RESOLVED from RESEARCH.md Open Question 3):**

1. Create a private generic helper:
```rust
fn eval_partial_inner<const N: usize>(
    &self, input: &[f64], output: &mut [f64],
    vars: VarType, order: u32,
) -> Result<(), XcError>
```

2. Inside the helper, implement the variable-pair loop from `docs/design/05-processing-flows.md`:
   - For order 0: evaluate once with CTaylor<f64, 1> (just the constant), output[0] = energy
   - For order 1: seed each input variable k with `c[0] = input[k], c[1<<k] = 1.0`, evaluate once, extract c[0] (energy) and c[1<<k] for each k (first derivatives)
   - For order 2: iterate over pairs (i,j) with i<=j. For each pair, create CTaylor inputs seeded for variables i and j (for i==j: seed variable i in two independent "slots"; for i!=j: seed variables i and j). The resulting coefficient at the mixed index gives d^2E/dx_i dx_j.
   - The output mapping follows taylorlen ordering: [E, dE/dx0, dE/dx1, d2E/dx0^2, d2E/dx0dx1, d2E/dx1^2] for 2 vars at order 2 (6 elements).

3. Dispatch on n_vars to select the const generic N:
```rust
match n_vars {
    1 => self.eval_partial_inner::<2>(input, output, vars, order),
    2 => self.eval_partial_inner::<4>(input, output, vars, order),
    3 => self.eval_partial_inner::<8>(input, output, vars, order),
    4 => self.eval_partial_inner::<16>(input, output, vars, order),
    5 => self.eval_partial_inner::<32>(input, output, vars, order),
    6 => self.eval_partial_inner::<64>(input, output, vars, order),
    7 => self.eval_partial_inner::<128>(input, output, vars, order),
    _ => return Err(XcError::InvalidOrder { order, mode: EvalMode::PartialDerivatives, n_vars }),
}
```

4. The key insight: CTaylor<f64, N> provides ALL first-order information in a single evaluation. For second-order, you iterate over variable pairs and re-seed. The C++ test data for SlaterX at order 2 with 2 vars (A_B) has 6 output elements: [E, dE/da, dE/db, d2E/da2, d2E/dadb, d2E/db2]. The first 3 come from a single evaluation (order 1). The last 3 come from the pair loop (order 2). Read `docs/design/05-processing-flows.md` for the exact seeding and extraction protocol.

**Potential mode** (per D-12, EVAL-03):
For LDA (Dependency::DENSITY only), potential is simple:
- Seed CTaylor<f64, N> with unit derivative on density variable (N = 1 << n_vars)
- output[0] = energy (c[0])
- output[1] = dE/d(alpha) (c[1]) -- this IS v_alpha for LDA
- For spin-polarized (A_B): output[2] = dE/d(beta) = c[2]

**Contracted mode** (per EVAL-04):
Input contains pre-computed Taylor coefficients. Create CTaylor from input directly (copy coefficients into c array), evaluate functionals, extract all coefficients to output.
