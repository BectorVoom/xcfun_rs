## Critical Anti-Patterns (do NOT repeat these)

- ANTI-PATTERN: **delegating Task-7.3-style long sweeps to worktree subagents** → run inline in main working tree with `Bash run_in_background=true` + paired stall-detector Monitor.
- ANTI-PATTERN: **assuming `cargo run -p validation` writes incrementally** → it doesn't; budget enough wall-clock for clean exit, or scope down with `--filter`.
- ANTI-PATTERN: **assuming `timeout 3000s` is enough for a full order-2 sweep** → it isn't. Real budget is 75-115 min for ~76 functionals; use 5400-7200s (90-120 min) cap.
- ANTI-PATTERN: **trusting an `exit 0` from `cargo run | tee` pipelines as "the binary completed cleanly"** → tee returns 0 even if the upstream binary was timeout-killed. Check `report.jsonl` mtime/size to confirm the binary actually wrote output.
