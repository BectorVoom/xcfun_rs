<objective>
Land the `release.yml` GitHub Actions workflow with the sdist + 3-platform wheel build
matrix per PY-06 + D-15 wheel side. Each platform leg builds the wheel via
`PyO3/maturin-action@v1`, INSTALLS the wheel into the runner's Python, and runs
`pytest crates/xcfun-py/tests/ -q` against the installed wheel — verifying the wheel
ships everything pytest needs (eval_parity.json fixture, type stubs, `__init__.py`
shim, and the cdylib).

This plan covers ONLY the wheel + pytest legs of `release.yml`. The PyPI publish job
(OIDC trusted publisher) and the GitHub Release artifacts job (`libxcfun_capi.{so,dylib,dll}`
+ cbindgen header) land in Plan 07-09 — both are appended to the SAME `release.yml`.

The xtask publish driver (D-15) lives in Plan 07-08 (separate Rust binary, runnable
locally pre-tag).

Pitfall 5 (yank-irreversibility): the wheel build MUST install + pytest from the wheel
on each platform BEFORE any tag is pushed. The `release.yml` workflow can be triggered
on a feature-branch push (workflow_dispatch) for dry-run validation; only tag pushes
trigger the publish jobs (added in 07-09).

Output: a `.github/workflows/release.yml` file with sdist + 3 wheel jobs + per-job
install + pytest verification. Tag-push trigger is wired but the publish + release-artifacts
jobs are placeholders / commented (Plan 07-09 fills them).
</objective>

    ```yaml
    # .github/workflows/release.yml
    #
    # Phase 7 Plan 07-07 — sdist + 3-platform wheel matrix + pytest-from-wheel.
    # Plan 07-09 appends publish-pypi (OIDC) + release-artifacts + github-release.
    #
    # Trigger:
    #   - push: tags matching v* (release path; publish jobs run after Plan 07-09 lands)
    #   - workflow_dispatch: manual dry-run from any branch (no publish; just builds)
    #
    # Sources:
    #   - 07-RESEARCH Example E lines 962-1115 (verbatim template)
    #   - github.com/PyO3/maturin-action README hardening guide
    #   - docs.pypi.org/trusted-publishers (used by Plan 07-09)

    name: Release

    on:
      push:
        tags:
          - 'v*'
      workflow_dispatch:

    env:
      PACKAGE_NAME: xcfun_rs
      # abi3-py310 — ONE wheel covers Python 3.10/3.11/3.12/3.13 per platform.
      PYTHON_VERSION: "3.10"

    jobs:
      sdist:
        name: Build sdist
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v6
            with:
              submodules: recursive
              fetch-depth: 0
          - uses: actions/setup-python@v6
            with:
              python-version: "${{ env.PYTHON_VERSION }}"
          - name: Build sdist
            uses: PyO3/maturin-action@v1
            with:
              command: sdist
              args: --out dist --manifest-path crates/xcfun-py/Cargo.toml
          - uses: actions/upload-artifact@v7
            with:
              name: wheels-sdist
              path: dist

      linux:
        name: Build + test wheel — linux x86_64
        runs-on: ubuntu-22.04
        steps:
          - uses: actions/checkout@v6
            with:
              submodules: recursive
              fetch-depth: 0
          - uses: actions/setup-python@v6
            with:
              python-version: "${{ env.PYTHON_VERSION }}"
          - name: Build wheel (manylinux_2_28_x86_64)
            uses: PyO3/maturin-action@v1
            with:
              target: x86_64-unknown-linux-gnu
              manylinux: "2_28"
              args: >-
                --release
                --out dist
                --manifest-path crates/xcfun-py/Cargo.toml
                --features cpu
          # Pitfall 7 — pytest MUST run against the INSTALLED wheel, not the
          # source tree, to catch wheel-packaging regressions (missing py.typed,
          # wrong abi3 tag, missing fixtures).
          - name: Install + pytest-from-wheel
            run: |
              python -m pip install --upgrade pip pytest numpy
              pip install dist/${{ env.PACKAGE_NAME }}-*.whl --force-reinstall
              python -c "import xcfun_rs; print('xcfun_rs', xcfun_rs.version())"
              pytest crates/xcfun-py/tests/ -q
          - uses: actions/upload-artifact@v7
            with:
              name: wheels-linux
              path: dist

      macos:
        name: Build + test wheel — macos arm64
        runs-on: macos-latest
        steps:
          - uses: actions/checkout@v6
            with:
              submodules: recursive
              fetch-depth: 0
          - uses: actions/setup-python@v6
            with:
              python-version: "${{ env.PYTHON_VERSION }}"
          - name: Build wheel (macosx_11_0_arm64)
            uses: PyO3/maturin-action@v1
            with:
              target: aarch64-apple-darwin
              args: >-
                --release
                --out dist
                --manifest-path crates/xcfun-py/Cargo.toml
                --features cpu
          - name: Install + pytest-from-wheel
            run: |
              python -m pip install --upgrade pip pytest numpy
              pip install dist/${{ env.PACKAGE_NAME }}-*.whl --force-reinstall
              python -c "import xcfun_rs; print('xcfun_rs', xcfun_rs.version())"
              pytest crates/xcfun-py/tests/ -q
          - uses: actions/upload-artifact@v7
            with:
              name: wheels-macos
              path: dist

      windows:
        name: Build + test wheel — windows x86_64
        runs-on: windows-latest
        steps:
          - uses: actions/checkout@v6
            with:
              submodules: recursive
              fetch-depth: 0
          - uses: actions/setup-python@v6
            with:
              python-version: "${{ env.PYTHON_VERSION }}"
          - name: Build wheel (win_amd64)
            uses: PyO3/maturin-action@v1
            with:
              target: x86_64-pc-windows-msvc
              args: >-
                --release
                --out dist

            shell: bash

              pytest crates/xcfun-py/tests/ -q
          - uses: actions/upload-artifact@v7
            with:
              name: wheels-windows
              path: dist

      # ---- Plan 07-09 appends below ---------------------------------------------------
      # publish-pypi:        OIDC trusted publisher (pypa/gh-action-pypi-publish@release/v1)
      # release-artifacts:   per-platform `cargo build -p xcfun-capi --release` + cbindgen header
      # github-release:      attach release-artifacts + xcfun.h to the GH Release on tag push
    ```
