    ```yaml
      publish-pypi:
        name: Publish to PyPI (OIDC trusted publisher)
        # Tag-push only — no PyPI publish on workflow_dispatch / branch push.
        if: startsWith(github.ref, 'refs/tags/v')
        needs: [sdist, linux, macos, windows]
        runs-on: ubuntu-latest
        environment: pypi
        permissions:
          # Pitfall 6 — MUST be set at JOB level, not just workflow level.
          # Otherwise pypa/gh-action-pypi-publish reports "OIDC token not available".
          id-token: write
        steps:
          - uses: actions/download-artifact@v8
            with:
              pattern: wheels-*
              merge-multiple: true
              path: dist
          - name: Publish via OIDC trusted publisher
            uses: pypa/gh-action-pypi-publish@release/v1
            with:
              skip-existing: true
              packages-dir: dist
              verbose: true

      release-artifacts:
        name: Build libxcfun_capi for GH Release (D-16)
        if: startsWith(github.ref, 'refs/tags/v')
        needs: [linux, macos, windows]
        strategy:
          matrix:
            include:
              - os: ubuntu-22.04
                target: x86_64-unknown-linux-gnu
                artifact: libxcfun_capi.so
              - os: macos-latest
                target: aarch64-apple-darwin
                artifact: libxcfun_capi.dylib
              - os: windows-latest
                target: x86_64-pc-windows-msvc
                artifact: xcfun_capi.dll
        runs-on: ${{ matrix.os }}
        env:
          # D-16 — RUSTFLAGS empty (no -Cfast-math), preserved discipline.
          RUSTFLAGS: ""
        steps:
          - uses: actions/checkout@v6
            with:
              submodules: recursive
              fetch-depth: 0
          - name: Add Rust target
            run: rustup target add ${{ matrix.target }}
          - name: Verify RUSTFLAGS empty (D-16 lock)
            shell: bash
            run: |
              if [ -n "${RUSTFLAGS:-}" ]; then
                echo "RUSTFLAGS must be empty for D-16; got: ${RUSTFLAGS}"
                exit 1
              fi
          - name: Build xcfun-capi (release)
            run: cargo build --release -p xcfun-capi --target ${{ matrix.target }}
          - name: Regenerate cbindgen header
            # Re-runs the xtask drift gate (Phase 5 D-09) to ensure xcfun.h is
            # fresh for the GH Release attachment.
            run: cargo run -p xtask --bin regen-capi-header --release
          - name: Stage release artifacts
            shell: bash
            run: |
              mkdir -p release-artifacts
              cp target/${{ matrix.target }}/release/${{ matrix.artifact }} release-artifacts/ || \
                cp target/${{ matrix.target }}/release/$(echo ${{ matrix.artifact }} | sed 's/^lib//' | sed 's/\.so$/.so/' | sed 's/\.dylib$/.dylib/') release-artifacts/ || true
              cp crates/xcfun-capi/include/xcfun.h release-artifacts/
          - uses: actions/upload-artifact@v7
            with:
              name: capi-${{ matrix.target }}
              path: release-artifacts/

      github-release:
        name: Attach artifacts to GitHub Release
        if: startsWith(github.ref, 'refs/tags/v')
        needs: [release-artifacts, publish-pypi]
        runs-on: ubuntu-latest
        permissions:
          # MUST be at JOB level (Pitfall 6 same shape as publish-pypi).
          contents: write
        steps:
          - uses: actions/checkout@v6
          - uses: actions/download-artifact@v8
            with:
              pattern: capi-*
              merge-multiple: true
              path: artifacts
          - name: Create / update GitHub Release with attachments
            uses: softprops/action-gh-release@v2
            with:
              # Files to attach to the GH Release. The release entry itself is
              # created/updated by the action when a tag push is detected.
              files: |
                artifacts/libxcfun_capi.so
                artifacts/libxcfun_capi.dylib
                artifacts/xcfun_capi.dll
                artifacts/xcfun.h
              generate_release_notes: true
              # CHANGELOG.md (Plan 07-10) supplies the body; generate_release_notes
              # appends auto-extracted PR / commit notes for visibility.
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → PyPI | OIDC short-lived token (no long-lived secret) |
| GitHub Actions runner → GitHub Release | `contents: write` permission scoped to the release-attach job |
| Tag push (refs/tags/v*) → release pipeline | Only authenticated push to refs/tags/v* triggers the publish path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-09-01 (T-7-V2) | Spoofing | Stolen long-lived API token uploads malicious wheel | mitigate | OIDC trusted publisher (no MATURIN_PYPI_TOKEN secret); rotates per workflow run |
| T-7-09-02 (T-7-V4) | Privilege escalation | Publish job inheriting workflow-level permissions broader than needed | mitigate | `permissions: id-token: write` set at JOB level on publish-pypi only; `permissions: contents: write` on github-release only |
| T-7-09-03 (T-7-V14) | Tampering | -Cfast-math leakage in libxcfun_capi build via accidental RUSTFLAGS env override | mitigate | release-artifacts step verifies RUSTFLAGS is empty (Step 3 in workflow) |
| T-7-09-04 (T-7-V14) | Tampering | xcfun.h drifting from upstream `xcfun-master/api/xcfun.h` | mitigate | regen-capi-header runs in release-artifacts; Phase 5 headers-match CI gate (existing) catches drift |
| T-7-09-05 | Repudiation | Tag pushed by unauthorized actor triggers publish | mitigate | `if: startsWith(github.ref, 'refs/tags/v')` + repo branch protection rules (operator-managed; not in workflow) |
